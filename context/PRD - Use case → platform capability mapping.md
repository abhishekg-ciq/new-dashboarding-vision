# **Use case → platform capability mapping**

## **Open questions**

* **Semantic layer registration:** Must FDE-created metrics register in the semantic layer before they’re queryable? (Rec: yes — prevents Flask-app divergence.)

* **Skills definition:** Are skills metric definitions (light, semantic layer handles) or callable Python/SQL functions (heavy, platform eng)? For FDEs or end users?

* **Tier 3 role gating:** Who sees the full edit/create panel? Per-account flag (acceptable) or per-user role (ideal)? Need user-role data we don’t fully have.

* **MCP moat:** If data is MCP-exposed, moat \= proprietary intelligence (RCA, alerts, agentic actions) \+ writeback. Not raw data.

* **POC evaluation criteria:** Claude SDK & UI MCP POCs need rubric: (1) semantic layer consumption, (2) FDE configurability, (3) CIQ design, (4) Target spec, (5) data governance.

* **Edit panel removal:** Rec: do NOT remove. 12 power-creator combos (Newell, Emerson, Lego, Meta, etc.) actively build. Demote, don’t delete.
